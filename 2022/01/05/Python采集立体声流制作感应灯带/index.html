环境需要

PC+ESP8266

  • Python的版本＞3.5 (安装Anaconda 配置 Windows下的python环境)

  • 引出 RX1 针脚的ESP8266-WIFI模块即可 .

    ​ 以下模块也可代替:

    • NodeMCU v3
    • Adafruit HUZZAH
    • Adafruit Feather HUZZAH

  • 5V 电源

  • USB 音频输入接口. 连接USB麦克风或者声卡.
  • WS2812B LED 灯带
  • 5V电源

硬件连接

ESP8266

​ ESP8266支持 I²S 协议(国内暂时没有相关词条，只给出官方文档进行解释)控制 ws2812b LED 灯带. LED灯带 **一定 **通过 RX1pin连接

​ ESP8266模块上的RX1引脚应连接至ws2812b LED条（通常标记为DIN或D0）的数据输入引脚。

树莓派

​ 使用树莓派时由于树莓派是3.3v供电，所以使用5v电源供应时需要降压模块。同时，树莓派的GPIO不能直接对灯带供电，需要对灯带添加额外的5v电源

连接方式：

  • 灯带和树莓派需要共地，将两个GND连接
  • 5v电源连接灯带的正极
  • 将树莓派上的PWM GPIO引脚连接至LED条上的数据引脚。如果使用树莓派的2或3，则尝试引脚18（GPIO5）。

配置文件

  1. 安装Python库
  2. 在arduino中安装ESP8266的依赖库(参考上文)
  3. 将ESP8266模块的RX1引脚连接至ws2812b LED灯带的数据输入引脚。确保LED灯带正确连接到5V电源，并且ESP8266和LED条共用一个公共地线。
  4. 修改注释部分即可 ws2812_controller.ino:
  • 修改const char* WIFI名称 为你连接的WIFI名称
  • 修改 const char* 密码 为你的WIFI密码
  • 修改 IPAddress gateway 你的网关地址
  • 修改IPAddress ip 为你的IP地址
  • 修改#define NUM_LEDS 为你的LED灯的数量
  1. 编译并上传 ws2812_controller.ino 到 ESP8266. 确保在Arduino IDE菜单中选择正确的开发板类型。 在下拉菜单中，将“CPU频率”设置为160 MHz以获得最佳性能。
  2. 在 config.py 中:
  • 设置 N_PIXELS 你的灯柱数量 （要和上文匹配）
  • 修改UDP_IP 你的ESP8266的IP地址 （要和上文匹配）
  • 如果需要，将“麦克风频率”设置为您的话筒采样频率，单位为Hz。一般不用修改。

测试 LED 灯带

  1. cd rpi_ws281x/python/examples
  2. sudo nano strandtest.py
  3. 配置文件顶部的选项。如果使用的是反转逻辑电平转换器，则启用逻辑反转。为LED灯带设置正确的GPIO引脚和像素数。可能需要一个逻辑电平转换器来将树莓派的3.3V逻辑转换为ws2812b LED灯带使用的5V逻辑。
  4. 调试例程 ： ‘sudo python strandtest.py’

音频输入

​ 可视化程序从默认音频输入设备（由操作系统设置）传输音频。

音源:

  • 音频电缆连接到音频输入插孔（需要Raspberry Pi上的USB声卡）
  • 网络摄像头麦克风、耳机、录音室录音麦克风等

虚拟音频源

您可以使用“虚拟音频设备”将音频播放从一个应用程序传输到另一个应用程序。这意味着您可以在计算机上播放音乐，并将播放直接连接到可视化程序。

Windows

​ 在Windows上，您可以使用“立体声混音”将音频输出流复制到音频输入中。立体声混音仅支持某些音频芯片组。如果您的芯片组不支持立体声混音，您可以使用第三方应用程序，如Voicemeeter.

​ 转到Windows声音设置下的录制设备（控制面板->声音）。在右键单击菜单中，选择“显示禁用的设备”。

​ 启用立体声混音并将其设置为默认设备。您的音频播放现在应该用作可视化程序的音频输入源。如果您的音频芯片组不支持立体声混音，则它将不会出现在列表中。

可视化GUI控制

​ 配置好所有环境后，运行 visualization.py 。随后可视化GUI 运行，它会实时记录麦克风和扬声器的信号并转换后通过WIFI发送到ESP8266

​ 如果你想使用GUI的默认配置的话，可以打开 config.py 设置是否自动运行

注意事项

  • ESP8266 支持最多 256 个LED灯点. 树莓派可以设置任意数量
  • 仅支持偶数个像素块设置。
