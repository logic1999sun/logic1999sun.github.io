如何优雅使用实验室深度学习服务器

​ 很久没有动手写博文了，一是最近没有可以分享的项目给大家，二是很难有闲做一些小玩意娱乐一下（没错就是我鸽了很久的FPV）。但是今天要分享的是如何优雅的使用实验室服务器，起因是最近在学习深度学习相关的东西，经常在自己的laptop上码好，然后放到实验室的工作站上训练，做到代码同步很困难。当下对于远程办公的需求逐步增加，同时一位师兄也要跑相关的东西，但我俩一个在windows一个在Linux……

​ 根据以上情况得出两个需求：

  1. 我需要远程code，随时随地不受局域网限制启用远程服务
  2. 由于是多人在用，需要对服务器分配资源

经过这几天的学习，探索出一套方案：内网穿透+云服务器+pycharm同步+LXD/Docker

由于服务器大多数的用户只有两人所以最后放弃的LXD多人管理的方案，随即转入Docker的怀抱。但是如果你的资源很多，用户也很多，并且拥有一个UI界面，首推LXD.

安装Ubuntu

​ 因为我是双系统安装，这一段也是有很多坑所以还是有必要详细写一下的

准备阶段

  1. 一个容量4G以上U盘作为系统启动盘
  2. 腾出足够的硬盘空间，为系统资源配置留有余地（双系统要提前准备好，后期再挂载太繁琐了）
  3. 进入官网选择系统版本进行下载镜像文件
  4. 我使用的是rufus来制作启动盘

系统分区

安装步骤大同小异，重中之重便是双系统下的系统分区问题

  1. efi引导分区，作为ubuntu的启动项

    大小：1024mb

    类型：逻辑分区

    位置：空间起始位置

    用于：EFI系统分区

  2. swap 交换空间，用于虚拟内存

    大小：根据物理内存设定

    类型：主分区

    位置：空间起始位置

    用于：交换空间

  3. 挂载点 / ,存放ubuntu系统文件

    大小：根据需求和容量自定义

    类型：逻辑分区

    位置：空间起始位置

    用于：EXT4日志文件系统

    挂载点：/

  4. 挂载点 /usr ，存放用户程序

    大小：根据需求和容量自定义

    类型：逻辑分区

    位置：空间起始位置

    用于：EXT4日志文件系统

    挂载点：/usr

  5. 挂载点 /home ,存放用户文件

    大小：根据需求和容量自定义

    类型：逻辑分区

    位置：空间起始位置

    用于：EXT4日志文件系统

    挂载点：/home

分区完成后，设置安装启动器设备，选择EFI分区，这样系统分区就完成啦

远程桌面

Xdrp

  1. sudo apt install xrdp
  2. sudo systemctl status xrdp
  3. sudo adduser xrdp ssl-cert
  4. sudo systemctl restart xrdp
  5. 以上即可使用xrdp远程桌面服务

VNC

  1. sudo apt install tightvncserver
  2. 使用vncserver创建端口
  3. 以上就可访问工作站图形界面

LXD配置

安装LXD

初始化LXD

接下来和命令行交互，选择选项。

  • LXD Clustering：不需要
  • New storage pool：需要创建一个存储池
  • Name of storage pool：给存储池命名
  • storage backend：存储后端，使用 ZFS
  • Create a new ZFS pool：需要创建一个 ZFS 池
  • Use an existing block device：No
  • Size in GB of new loop device 预分配多大的空间作文 zfs 后端:
  • MAAS server?：裸金属服务器，不需要
  • New local network bridge?：需要，我只需要使用 LXD 默认的网桥即可
  • New bridge be called：给网桥命名
  • IPv4：默认 auto
  • IPv6：默认 auto
  • LXD available over the network？：默认 no
  • Stale cached？：默认 yes
  • YAML printed？：打印信息，yes/no 都行
